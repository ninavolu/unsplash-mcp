How it works

Simple by design.

One URL to paste. The server runs in the cloud — nothing to install locally.

01

Paste the URL

Add the server's Railway URL to your Claude MCP settings. One JSON field — no API key or account needed on your end.

02

Ask Claude

Type "find me a photo of a coffee shop" and Claude searches Unsplash through the MCP tool and returns results instantly.

03

Use with credit

Every photo includes a ready-to-paste attribution_html — legally compliant with Unsplash API guidelines.

Setup guide

Up and running in 2 minutes.

Follow these steps in order. If anything looks off, open an issue on GitHub.

  1. 1

    Open Claude MCP settings

    In the Claude desktop app go to Settings → Developer → Edit Config. This opens a JSON file.

    ℹ

    Using Claude Code? Run claude mcp add from your terminal instead and follow the prompts.

  2. 2

    Add the connector entry

    Paste this inside your "mcpServers" object. Replace the URL with your Railway deployment URL.

    ~/.claude.json 
    "unsplash": {
  "type": "http",
  "url":  "https://YOUR-RAILWAY-URL.railway.app/mcp"
}
    ⚠

    Replace YOUR-RAILWAY-URL with the URL from your Railway project. It looks like unsplash-mcp-production-xxxx.railway.app.

  3. 3

    Restart Claude and test

    Close and reopen the Claude app. Then try this prompt:

    Claude chat 
    Search Unsplash for a mountain lake at sunrise, landscape orientation.

    Claude will call search_photos and return results with attribution.

  4. 4

    Embed the photo in your content

    Use the returned fields directly in your HTML or markdown:

    your-page.html 
    <!-- regular URL is best for web -->
<img src="{photo.urls.regular}"
     alt="{photo.alt_description}">

<!-- paste attribution_html as-is -->
<p>{photo.attribution_html}</p>
Security & compliance

Built submission-ready.

Answers to common questions about auth, domains, and the Anthropic pre-submission checklist.

✓

Do I need an official domain?

No. A Vercel or Netlify subdomain counts as valid public documentation for the Anthropic directory. A custom domain is optional.

✓

Do users need their own API key?

No. The server runs as a shared proxy — your Unsplash key lives server-side. Users connect via URL only. Rate limiting (30 req/hour per IP) protects the shared quota.

✓

Tool annotations — required by Anthropic

All three tools include title, readOnlyHint, and destructiveHint annotations. Search tools are read-only; download tracking is marked as having side effects.

✓

No prompt-injection in tool descriptions

Descriptions state only what each tool does — no instructions to Claude about when to call them, no behavioral overrides.

!

Test credentials — required for reviewers

Before submitting, you must provide a working Unsplash API key to Anthropic reviewers so they can test every tool. Get one at unsplash.com/developers.

🛡

IP rate limiting

30 requests per IP per hour in memory. Prevents any single user from exhausting the shared Unsplash quota. Resets on server restart.

🔑

Key isolation

Your Unsplash API key is stored only as a Railway environment variable. It never appears in responses or logs.

📡

HTTPS only

Railway terminates TLS automatically. All traffic between Claude and the server is encrypted in transit.

🧹

Stateless server

No database. No user data stored to disk. The only state is the in-memory rate limit counter, which disappears on restart.

Privacy policy

We collect nothing personal.

Your search queries go to Unsplash. Nothing is stored on our end.

What we collect

  • Nothing that identifies you
  • No account, login, or email required
  • Transient in-memory rate limit counter per IP (resets hourly)

What Unsplash collects

Search queries are forwarded to the Unsplash API. Their own privacy policy governs that data.

Data storage

No database. No disk writes. The server is stateless — rate limit state lives in memory and disappears on every restart.

Third parties

Hosted on Railway. Railway may retain infrastructure-level logs per their own policy. We share no user data with any other party.

Full Privacy Policy

Last updated: June 2, 2026

1. Information we collect

This server does not collect, store, or process any personally identifiable information. No account registration, email address, name, or identifying data is required to use this service. The only transient data retained is a rate-limit counter keyed to your IP address, held in memory only, never written to disk, and reset on server restart.

2. How your data is used

Search queries sent through MCP tools are forwarded directly to the Unsplash API to retrieve photo results. These queries are not logged, analyzed, or stored by this service. Unsplash's handling of that data is governed by their privacy policy at unsplash.com/privacy.

3. Cookies and tracking

This service sets no cookies and uses no tracking pixels, analytics scripts, or fingerprinting of any kind.

4. Data retention

No user data is retained. The server is stateless beyond the in-memory rate-limit counter described above.

5. Third-party services

  • Unsplash API — photo search results. Subject to Unsplash's privacy policy.
  • Railway — cloud hosting provider. May retain infrastructure logs per their own policy.

6. Children's privacy

This service is not directed at children under 13 and does not knowingly collect data from minors.

7. Changes to this policy

Updates will be reflected on this page with a revised date. Continued use constitutes acceptance of the updated policy.
